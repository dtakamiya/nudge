# 1on1 シミュレーション レポート

**実施日:** 2026-02-23
**対象ブランチ:** main
**対象バージョン:** Next.js 16.1.6 + Prisma 6.19.2

---

## 実施内容

ブラウザ（Chrome DevTools Protocol）を用いて、以下の一連の 1on1 フローをシミュレーション実施。

1. ダッシュボードを確認
2. メンバー「佐野 友美」を選択
3. 1on1 準備ページでアジェンダを入力
4. ミーティング新規作成フォームで各項目を入力・保存

---

## 発見した問題点

### 🔴 高優先度

#### 1. DB マイグレーション未適用によるエラー

**ページ:** `/members/{id}/meetings/prepare`
**現象:** ページを開いた際に 500 エラーが発生。

```
PrismaClientKnownRequestError: The column `main.Meeting.mood` does not exist in the current database.
```

**原因:** `add_mood_to_meeting` 等の 7 件のマイグレーションが未適用のまま本番 DB に対してクエリが実行された。

**対処:** `npx prisma migrate deploy` を実行して解消。
**再発防止策:** 開発環境セットアップ手順に `npx prisma migrate deploy` を明記する。

---

#### 2. ミーティング保存が機能しない（要検証）

**ページ:** `/members/{id}/meetings/new`
**現象:**

- フォームを入力して保存ボタンを押すと `ClosingDialog` が表示される
- ダイアログ内の「保存する」または「このまま保存」を押しても画面が遷移しない
- DB にミーティングレコードが作成されない
- サーバーログ上は `POST 200` が複数回発生している

**調査結果:**

`ClosingDialog` には 2 つの表示パターンがある:

| 状態                                 | 表示ボタン                           | 条件                     |
| ------------------------------------ | ------------------------------------ | ------------------------ |
| アクションなし (`hasNoAction=true`)  | 「追加してから保存」「このまま保存」 | actionItems がすべて空   |
| アクションあり (`hasNoAction=false`) | 「戻る」「保存する」                 | 1 件以上有効なアクション |

シミュレーション時は `hasNoAction=true` の状態（アクションタイトルが React state に反映されていない）。「このまま保存」クリック後も保存されなかった。

**疑われる原因:**

- `createMeeting` Server Action が `{success: false}` を返しているがエラー表示なし
- バリデーションエラー（topics の title が空など）で silent fail している可能性

**確認すべきこと:** `createMeeting` の戻り値チェックとエラーメッセージ表示の実装。

---

### 🟡 中優先度

#### 3. テンプレート選択の視覚的フィードバックなし

**ページ:** `/members/{id}/meetings/prepare`
**現象:** 「定期チェックイン」「キャリア面談」等のテンプレートボタンをクリックしても、選択状態を示す視覚的な変化（枠線・背景色等）がない。

**期待する動作:** 選択中のテンプレートが強調表示される。

**影響:** ユーザーがどのテンプレートを選んでいるか判別できない。

---

#### 4. 「話題を追加」ボタンのクリック反応が不安定

**ページ:** `/members/{id}/meetings/prepare`
**現象:** ボタンをクリックしても話題入力フィールドが追加されないケースがある。

**調査結果:** React の合成イベント（`onClick`）が、ネイティブの DOM クリックイベントと異なるイベントパイプラインを使うため、特定の状況でイベントが発火しない可能性がある。

---

#### 5. アクション追加後の入力状態が保存に反映されない

**ページ:** `/members/{id}/meetings/new`
**現象:** 「アクション追加」ボタンでフィールドを追加しテキストを入力しても、保存時に空として扱われる。

**原因:** React controlled input への値セットに `onChange` 合成イベントが必要だが、直接 `value` を書き換えると React state が更新されない。
**ユーザー操作の場合:** 通常のキー入力では問題なく動作するはず。アプリ側のバグではない可能性が高い。

---

### 🟢 軽微・改善提案

#### 6. チャートの色が黒で表示される

**ページ:** `/analytics`, `/members/{id}`
**現象:** 棒グラフのバーが黒く表示される。

**原因:** SVG のプレゼンテーション属性に `hsl(var(--primary))` を指定しても CSS カスタムプロパティが解決されない。
**修正方針:** `hsl(var(--primary))` → `var(--primary)` に変更（CSS 変数の値がすでに有効な色値のため）。

※ユーザー判断により今回は修正対象外。

---

#### 7. ClosingDialog のボタンラベルが直感的でない

**現象:** アクションなし状態のとき「追加してから保存」「このまま保存」という 2 択が表示されるが、前者を選んでも「ダイアログを閉じてフォームに戻る」だけで、アクション追加を強制するわけではない。

**提案:** ボタンラベルを「戻って追加する」「アクションなしで保存」等、意図が明確な表現に変更する。

---

#### 8. チェックイン気分セレクターのラベル未表示

**ページ:** `/members/{id}/meetings/new`
**現象:** 気分を選ぶ絵文字ボタンのセクションにラベルや説明がなく、何を選んでいるのかがわかりにくい。
